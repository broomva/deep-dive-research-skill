# Contributing

Contributions are welcome! Here's how to contribute to the Deep-Dive Research Orchestrator skill.

## Ways to Contribute

### Report Bugs
- Found an issue? Open a GitHub issue with details
- Include the research request that triggered the bug
- Describe what you expected vs. what happened

### Suggest Improvements
- Have an idea for a better feature?
- Think the methodology could be improved?
- Open an issue with your suggestion

### Add Documentation
- Improve existing documentation
- Add examples or use cases
- Create guides or tutorials
- Fix typos or clarity issues

### Enhance the Skill
- Add support for new research dimensions
- Improve research quality
- Add new output formats
- Optimize execution time

## Process

### 1. Fork the Repository
```bash
git clone https://github.com/YOUR-USERNAME/deep-dive-research-skill
cd deep-dive-research-skill
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow existing code/documentation style
- Add tests or examples if applicable
- Update documentation as needed

### 4. Commit Your Changes
```bash
git commit -m "Brief description of changes"
```

### 5. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 6. Open a Pull Request
- Describe what you changed and why
- Reference any related issues
- Follow the PR template

## Contribution Guidelines

### Code Style
- Keep documentation clear and concise
- Use consistent formatting
- Follow existing patterns

### Documentation
- Update README if adding features
- Add examples for new capabilities
- Keep METHODOLOGY.md current
- Document any new research dimensions

### Testing
- Test your changes thoroughly
- Try different research requests
- Verify output quality
- Check for edge cases

### Examples
- Add example research requests for new features
- Include expected output examples
- Document any special considerations
